Guidelines for using the key building blocks exposed by this starter.

## Button

- Sizes: `sm | md | lg` (mapped to MUI).
- Variants: `contained | outlined | text`.
- No automatic uppercasing (`textTransform: none`).

**Do**

- Use concise labels (“Save”, “Continue”).
- Prefer one primary action per view.

**Don’t**

- Stack many primary buttons side-by-side.
- Use color to convey state without text or icon.

## Form Primitives

- Use `RHFForm`, `FormTextField`, and `FormMessage` for consistent validation.
- Zod + `@hookform/resolvers/zod` is set up in examples.

## DataTable

- Light/Heavy lists: for light datasets, pass all rows; for heavy datasets, control with `{ totalRows, page, pageSize, onPageChange }`.
- Sorting: use `sort="col:asc,col2:desc"` and `onSortChange(...)` to keep URL/state in sync.
- Row actions: use `actionsColumn({ onEdit, onDelete })`.

## Templates

- `ListPage`: header with actions; optional filters bar; single table below.
- `CrudPage`: integrated edit drawer; supports **external** `onCreate` modals.

## Accessibility

- Reachable tab order, visible focus ring.
- Announce errors next to inputs; buttons must have accessible names (`aria-label` on icon-only).
